# Research Cache

> Incremental research findings that persist across compaction and sessions.
> Agents: UPDATE entries, don't recreate from scratch. Add new sections, update existing.
> Pre-compact hook saves context here. Session-start restores from here.

## How to Use

### Writing (agents):
- After completing research-first protocol, append findings here under a dated section
- If a section for the same topic exists → UPDATE it (merge new findings), don't duplicate
- Keep entries concise: file paths, key findings, decisions made
- Delete stale entries when the related work is completed and committed

### Reading (agents):
- Check this file BEFORE running fresh research — the answer may already be here
- After compaction: this file IS your research context. Read it first.
- If an entry looks stale (references deleted files, old branches) → delete it

## Format

```
## [YYYY-MM-DD] Topic: [brief description]
**Files**: [paths read]
**Findings**: [key discoveries]
**Decisions**: [what was decided and why]
**Status**: active | completed | stale
```
