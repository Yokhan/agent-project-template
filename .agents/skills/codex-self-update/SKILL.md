# Codex Self Update

Use with `$codex-skill-maintenance` for skill changes.

## Process

1. Classify the correction: bug, knowledge gap, style, design disagreement, misunderstanding, process.
2. Save the immediate lesson if it is reusable.
3. Decide whether to update lesson, rule, skill, validator, or docs.
4. Keep template-owned and project-owned surfaces separate.
5. Validate that the new rule catches the original failure.
